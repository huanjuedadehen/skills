# YouTube 视频摘要

对 YouTube 视频字幕进行结构化摘要，帮助用户快速判断视频是否值得观看。

## 环境变量

- `YOUTUBE_SUMMARIZER_WORKSPACE`：工作目录，存放字幕和摘要。需用户在 `~/.claude/settings.json` 的 `env` 字段中配置。

其余参数（短视频阈值、分段时长、段间重叠）作为 `fetch_transcript.py` 的默认值存在，用户若需覆盖可在调用时附加对应 CLI flag（详见 `python fetch_transcript.py save --help`）。

## 执行流程

收到用户提供的 YouTube 链接后，按以下步骤执行：

### 第一步：获取并保存字幕

先检查 `$YOUTUBE_SUMMARIZER_WORKSPACE/transcripts/{video_id}.json` 是否已存在。如果已存在，直接使用本地字幕，不重新拉取。

如果不存在，对用户提供的每个 YouTube 链接，运行：

```bash
python ${CLAUDE_SKILL_DIR}/scripts/fetch_transcript.py save <YouTube链接> [--lang <语言代码>]
```

- `--lang` 可选，默认为 `en`。如果用户指定了语言或视频是中文的，使用对应语言代码（如 `zh`）。
- 如果获取失败，先用 `list` 子命令查看可用字幕语言，再用合适的语言代码重试：

```bash
python ${CLAUDE_SKILL_DIR}/scripts/fetch_transcript.py list <YouTube链接>
```

字幕会自动保存到 `$YOUTUBE_SUMMARIZER_WORKSPACE/transcripts/{video_id}.json`。

### 第二步：判断摘要策略

根据上一步 `save` 命令的输出判断：

- 输出包含「短视频，无需切段」→ 走「直接摘要」流程
- 输出包含「长视频，已切分为 N 段」→ 走「分段摘要 + 合并」流程

先检查 `$YOUTUBE_SUMMARIZER_WORKSPACE/summaries/{video_id}.md` 是否已存在。如果已存在，询问用户是否要重新生成。

### 第三步：生成摘要

#### 短视频：直接摘要

读取字幕文件内容，将所有字幕文本拼接后，按照以下要求生成摘要：

**摘要要求：**

1. 先判断视频类型（访谈、教程、Vlog、新闻评论、产品评测、演讲等），再据此调整摘要侧重点。
2. 保留观点归属，区分不同发言者，不要将 A 的观点归到 B 头上。
3. 不要编造字幕中没有的信息。
4. 如果字幕是自动生成的，可能存在识别错误，请根据上下文合理推断，但不要过度脑补。

**输出格式：**

```markdown
# [视频标题或主题]

## 基本信息
- **视频类型**: （访谈 / 教程 / Vlog / 新闻评论 / 产品评测 / 演讲 / 其他）
- **主题领域**: （如：科技、编程、商业、生活方式等）
- **字幕来源**: （人工上传 / 自动生成）
- **视频时长**: （从字幕数据计算）

## 一句话总结
用一句话概括这个视频的核心内容。

## 关键要点
用 3-7 个要点列出视频的核心信息，每个要点 1-2 句话。

## 详细内容
根据视频类型，选择最合适的方式展开：
- 访谈类: 列出讨论的主要话题，每个话题下注明各方观点，标注争议点和金句。
- 教程类: 列出教学目标、前置知识、关键步骤大纲和核心要点。
- Vlog 类: 按时间线梳理关键事件和节点。
- 评论/分析类: 提炼核心论点、支撑论据和最终结论。
- 产品评测类: 列出评测对象、优缺点和最终评价。
- 演讲类: 提炼主题、核心论点和关键例证。
如果视频混合了多种类型，选择最主要的类型来组织，必要时可以结合使用。

## 值得观看吗？
用 2-3 句话说明这个视频值得看的理由，或者可以跳过的理由。
```

#### 长视频：分段摘要 + 合并

第一步的 `save` 命令已经自动完成了切段并保存为独立文件。输出中会列出每段的文件路径，如：

```
长视频，已切分为 6 段:
  段 1: 0.0min - 10.0min (120 条字幕) -> /path/to/transcripts/{video_id}/seg_01.txt
  段 2: 9.0min - 20.0min (115 条字幕) -> /path/to/transcripts/{video_id}/seg_02.txt
  ...
```

**第一轮：逐段摘要**

依次处理每个段文件：

1. 用 Read 工具读取段文件（如 `seg_01.txt`）
2. 根据段内文本生成该段的摘要
3. 将段摘要保存到 `$YOUTUBE_SUMMARIZER_WORKSPACE/summaries/{video_id}/` 目录下，文件名与段文件对应（如 `seg_01_summary.md`）
4. 处理下一段，直到所有段完成

每段的摘要格式：

```markdown
### [起始时间 - 结束时间] 话题概述
用一句话概括这个时间段在聊什么。

### 要点
- 列出这段时间内的关键信息点，每个要点 1-2 句话
- 保持 2-5 个要点
- 如有重要观点或金句，标注发言者
```

分段摘要要求：
1. 保留观点归属，区分不同发言者。
2. 不要编造字幕中没有的信息。
3. 保持简洁，聚焦这个时间段内实际讨论的内容。
4. 如果这段内容是在延续上一段的话题，直接摘要当前段的内容即可，不需要交代前情。

**第二轮：合并摘要**

读取 `$YOUTUBE_SUMMARIZER_WORKSPACE/summaries/{video_id}/` 目录下所有段摘要文件，将内容合并后生成最终结构化摘要。

合并要求：
1. 去除段与段之间的重复内容（因为分段时有时间重叠）。
2. 理清话题脉络，将跨段的同一话题合并整理。
3. 保留观点归属，不要将不同发言者的观点混为一谈。
4. 不要编造分段摘要中没有的信息。

合并后的输出格式：

```markdown
# [视频标题或主题]

## 基本信息
- **视频类型**: （访谈 / 教程 / Vlog / 新闻评论 / 产品评测 / 演讲 / 其他）
- **主题领域**: （如：科技、编程、商业、生活方式等）
- **字幕来源**: （人工上传 / 自动生成）
- **视频时长**: （从字幕数据计算）

## 一句话总结
用一句话概括这个视频的核心内容。

## 关键要点
用 5-10 个要点列出视频的核心信息，每个要点 1-2 句话。

## 内容时间线
按时间顺序梳理视频的内容脉络：
- **[起始时间 - 结束时间] 话题标题**: 话题的核心内容概述，2-3 句话。

## 详细内容
根据视频类型，选择最合适的方式展开（同短视频规则）。

## 值得观看吗？
用 2-3 句话说明这个视频值得看的理由，或者可以跳过的理由。
```

### 第四步：保存摘要

- **短视频**: 将摘要保存到 `$YOUTUBE_SUMMARIZER_WORKSPACE/summaries/{video_id}.md`
- **长视频**: 最终合并摘要保存到 `$YOUTUBE_SUMMARIZER_WORKSPACE/summaries/{video_id}/final.md`（段摘要已在第三步保存）

保存完成后，向用户展示摘要内容，并告知保存路径。

## 批量处理

如果用户提供了多个 YouTube 链接，按顺序逐个处理，每个视频完成后展示摘要并告知进度（如「已完成 2/5」）。

## 注意事项

- 所有摘要输出使用中文。
